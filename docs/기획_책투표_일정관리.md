## 1. 현재 상태 분석

### ✅ 이미 구현된 기능
| 기능 | 설명 |
|------|------|
| **일정 투표 (polls)** | 참여/미참여 투표, 댓글, 마감 기한, 바 그래프 시각화 |
| **일정 확정** | 모임장이 투표 결과 기반으로 모임을 등록 → `meetings` 테이블 |

| 기능 | 설명 |
|------|------|
| **책 투표** | 다음 모임에서 읽을 책을 후보로 올리고 투표하는 기능 |
| **투표 히스토리** | 마감된 투표를 아카이브하여 과거 기록 열람 |
| **투표 → 모임 확정 자동 연결** | 투표 결과를 모임 확정에 자연스럽게 반영하는 흐름 |

## 2. 신규 기능 기획

### 2.1 📚 책 투표

독서모임에서 가장 핵심적인 "다음에 뭐 읽을까?"를 결정하는 투표 기능입니다.

#### 사용자 흐름

```
① 누구나 [책 투표 만들기] 버튼 클릭
② 투표 제목 입력 (예: "5월 모임 도서 선정")
③ 후보 도서 추가 (Google Books API로 검색 → 선택)
   - 최소 2권, 최대 5권
   - 표지, 제목, 저자, 페이지수 자동 표시
④ 마감 기한 설정
⑤ 투표 게시
```

```
⑥ 모임원들이 투표 카드에서 원하는 책에 투표
   - 1인 1표 (복수 선택 불가)
   - 투표 변경 가능 (마감 전까지)
⑦ 결과 실시간 표시 (막대 그래프 + 투표자 이름)
⑧ 마감 후 → 1위 도서 자동 하이라이트
⑨ 모임장이 [도서 확정] → 해당 모임의 선정도서로 자동 연결
```

#### 화면 구성

| 영역 | 내용 |
|------|------|
| **투표 헤더** | 투표 제목, 생성자, 마감일(D-day), 투표율(N/10명) |
| **후보 리스트** | 표지 썸네일 + 제목 + 저자 + 페이지수 + 득표수 |
| **투표 버튼** | 각 책 옆에 라디오 버튼 스타일 투표 |
| **미투표자** | 아직 투표하지 않은 모임원 이름 |
| **결과 바** | 각 책의 득표율 바 그래프 |
| **하단 액션** | 모임장: [도서 확정], 생성자: [수정/삭제] |

#### DB 설계

```sql
-- 책 투표 테이블
CREATE TABLE book_polls (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title TEXT NOT NULL,                -- "5월 모임 도서 선정"
  meeting_id UUID REFERENCES meetings(id),  -- 연결할 모임 (선택)
  created_by UUID NOT NULL REFERENCES members(id),
  deadline TIMESTAMPTZ,
  status TEXT DEFAULT 'active' CHECK (status IN ('active', 'closed', 'confirmed')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 후보 도서 테이블
CREATE TABLE book_poll_candidates (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  poll_id UUID NOT NULL REFERENCES book_polls(id) ON DELETE CASCADE,
  book_title TEXT NOT NULL,
  book_author TEXT,
  thumbnail TEXT,
  page_count INTEGER,
  description TEXT,
  added_by UUID REFERENCES members(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 책 투표 응답 테이블
CREATE TABLE book_poll_votes (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  poll_id UUID NOT NULL REFERENCES book_polls(id) ON DELETE CASCADE,
  candidate_id UUID NOT NULL REFERENCES book_poll_candidates(id) ON DELETE CASCADE,
  member_id UUID NOT NULL REFERENCES members(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(poll_id, member_id)  -- 1인 1표
);
```

### 2.2 🗓️ 모임일정 확정 개선

현재 일정 확정은 투표와 별도로 동작합니다. 투표 → 확정까지 자연스러운 흐름을 만듭니다.

#### 개선 포인트

| 현재 | 개선 후 |
|------|---------|
| 투표 마감 후 별도로 모임 등록 페이지 이동 | 투표 카드 안에서 바로 [이 일정으로 확정] 버튼 |
| 확정할 때 날짜를 다시 수동 입력 | 투표 내용의 날짜/시간/장소를 자동 채움 |
| 책 투표 결과와 모임이 연결 안 됨 | 확정 시 1위 도서도 함께 자동 등록 |

#### 모임장 전용 확정 흐름

```
① 일정 투표가 마감됨 (또는 모임장이 수동 마감)
② 투표 카드 하단에 [모임 확정하기] 버튼 표시 (모임장만)
③ 클릭 → 확정 모달:
   - 날짜/시간/장소: 투표 내용에서 자동 추출
   - 도서: 책 투표 1위 도서 자동 선택 (있는 경우)
   - 수정 가능
④ 확인 → meetings 테이블에 자동 등록
⑤ 참여 투표한 인원 → meeting_attendees 자동 등록
```

### 2.3 📋 신규 모임 등록 (직접 등록)

투표 없이 모임장이 직접 모임을 만드는 기능입니다. (현재 기능 유지 + 개선)

#### 개선 포인트

| 현재 | 개선 후 |
|------|---------|
| 날짜/시간만 입력 | 장소, 도서, 공지사항도 함께 입력 가능 |
| 여러 날짜 한번에 등록만 가능 | 단일 모임 등록도 간편하게 |

#### 등록 모달 필드

| 필드 | 필수 | 설명 |
|------|:----:|------|
| 날짜 | ✅ | date picker |
| 시간 | ✅ | 기본값 "오후 3시" |
| 장소 | ❌ | 텍스트 입력 |
| 도서 | ❌ | Google Books 검색으로 선택 |
| 공지사항 | ❌ | 텍스트 영역 |

### 2.4 📦 마감된 투표 히스토리

마감된 투표를 삭제하지 않고 아카이브하여 과거 기록을 보관하는 방법입니다.

#### 추천 방식: **접기(Collapse) + 아카이브 섹션**

> [!TIP]
> 별도 테이블 없이 기존 `polls` 테이블의 `deadline` 필드만으로 구현 가능합니다.

#### 화면 구성

```
┌─────────────────────────────────────┐
│  📊 진행중인 투표  (2개)              │
│                                     │
│  [투표 카드 A - 진행중]               │
│  [투표 카드 B - 진행중]               │
│                                     │
├─────────────────────────────────────┤
│  📦 지난 투표  ▸  (접힌 상태)         │ ← 클릭하면 펼쳐짐
│                                     │
│  (펼쳤을 때)                          │
│  📦 지난 투표  ▾  (3개)              │
│  [투표 카드 C - 마감됨] 읽기 전용      │
│  [투표 카드 D - 마감됨] 읽기 전용      │
│  [투표 카드 E - 마감됨] 읽기 전용      │
└─────────────────────────────────────┘
```

#### 구현 방식

| 항목 | 설명 |
|------|------|
| **분류 기준** | `deadline < 현재시간` → 마감된 투표 |
| **표시** | 마감 투표는 "지난 투표" 섹션에 접힌 상태로 표시 |
| **읽기 전용** | 마감 투표는 투표 버튼 비활성화, 결과만 열람 |
| **삭제 권한** | 모임장만 지난 투표 삭제 가능 |
| **DB 변경** | 없음 (기존 `deadline` 필드 활용) |
| **정렬** | 최신 마감순 (가장 최근 마감 투표가 위) |

#### 책 투표 히스토리도 동일 적용

- `book_polls.status`가 `closed` 또는 `confirmed`인 것도 "지난 투표" 섹션에 표시
- 확정된 도서는 `confirmed` 뱃지로 구분

## 3. 전체 사용자 흐름 (개선 후)

```
📋 일정 현황 페이지 (schedule)
 │
 ├── 📅 달력
 │
 ├── 다음 모임 (확정된 모임 목록)
 │
 ├── 📊 진행중인 투표
 │    ├── 일정 투표 카드 (참여/미참여)
 │    └── 📚 책 투표 카드 (후보 도서 투표)  ← NEW
 │         └── [도서 확정] → 모임에 자동 연결
 │
 ├── 📦 지난 투표 (접기/펼치기)  ← NEW
 │    ├── 마감된 일정 투표 (읽기 전용)
 │    └── 마감된 책 투표 (읽기 전용)
 │
 └── 하단 버튼
      ├── [일정 투표하기]
      ├── [📚 책 투표하기]  ← NEW
      └── [모임 등록하기] (모임장)
```
