# 1+1 독서모임 - 상세 기획서

## 1. 프로젝트 개요

**1+1 독서모임**은 10명의 모임원이 함께 일정을 조율하고, 도서를 선정하며, 발제문을 작성하고, 모임 기록을 남기는 올인원 독서모임 관리 웹앱입니다.

### 핵심 가치
- **빠른 결정**: 일정 투표를 통해 신속하게 모임 날짜 확정
- **직관적 UI**: 달력 형태로 한눈에 일정 현황 파악
- **체계적 관리**: 일정 → 도서 → 발제문 → 기록까지 단계별 관리

---

## 2. 사용자 및 권한

### 모임원 (총 10명)
오영준(모임장), 강다영, 김지원, 배성진, 이장민, 이경민, 홍다혜, 우동인, 한태원, 송의선

### 권한 구조

| 기능 | 모임장 | 일반 모임원 |
|------|--------|------------|
| 일정 제안 | ✅ | ✅ |
| 일정 투표 | ✅ | ✅ |
| 자신이 제안한 일정 삭제 | ✅ | ✅ |
| 일정 확정 | ✅ | ❌ |
| 도서 추천 | ✅ | ✅ |
| 발제문 작성/수정 | ✅ | ✅ |
| 모임 기록 작성/수정 | ✅ | ✅ |
| 모임원 추가/삭제 | ✅ | ❌ |
| 음성 녹음 | ✅ | ✅ |

> **권한 부여 방식**: 하드코딩된 이름 기반 로그인 (이름 선택 방식). 모임장 권한은 Supabase의 `members` 테이블에서 `role: 'leader'`로 관리.

## 3. 페이지 플로우 (UX)

```
메인 페이지 (일정 현황)
  ├── 달력 UI (제안된 일정 점 표시)
  ├── 모임 일정 투표 카드 목록
  │     ├── 참여 가능 / 불가능 숫자 표시
  │     ├── 직접 참여/불참 입력
  │     └── 제안자만 삭제 가능
  ├── 일정 제안 버튼 (누구나)
  └── 일정 확정 버튼 (모임장만)
        │
        ▼
  모임 상세 페이지
    ├── 확정된 날짜/시간
    ├── 도서 선정 (미선정 시 빈 상태 표시)
    ├── 발제문 / 질문 (미작성 시 빈 상태 표시)
    ├── 모임 기록 (미기록 시 빈 상태 표시)
    ├── AI 요약 (500자 이내, 마크다운)
    └── 음성 녹음 기능
```

### 페이지 단위 UX 흐름

1. **일정현황 메인페이지** → 달력 + 투표 카드
2. **투표** → 각 일정 안건에 참여/불참 투표
3. **일정확정** → 모임장이 확정 버튼 클릭
4. **도서 선정** → 누구나 책 추천 및 선정
5. **발제문 추천 및 제안** → 누구나 발제문/질문 작성
6. **모임 기록** → 음성 녹음 + 텍스트 기록 + AI 요약
7. **페이지 공유 및 확인** → 모든 모임원이 열람 가능

> 모든 페이지에 **뒤로가기 버튼** 포함

## 4. 주요 화면 설계

### 4.1 메인 페이지 (일정 현황)

- **상단**: "1+1 독서모임" 타이틀 + 현재 사용자 표시
- **달력 UI**: 
  - 월간 달력 형태
  - 제안된 일정이 있는 날짜에 점(dot) 표시
  - 확정된 일정은 다른 색상으로 표시
  - "오늘" 버튼으로 현재 날짜로 이동
- **모임 일정 투표 영역**:
  - 스크린샷 기준 초기 3개 안건:
    - 1안: 5월~8월 매월 첫번째 토요일 오후 3시 (참여 7, 불참 1)
    - 2안: 5월~8월 매월 첫번째 일요일 오후 3시 (참여 5, 불참 1)
    - 3안: 5월~8월 매월 두번째 토요일 오후 3시 (참여 3, 불참 1)
  - 각 안건에 참여/불참 숫자 표시
  - 제안자 이름 표시
  - 제안자만 삭제 가능 (삭제 버튼)
- **하단**: 다음 모임 / 선정된 도서 요약 (클릭 시 상세 페이지)

### 4.2 모임 상세 페이지

- 확정된 날짜, 시간, 장소
- 도서 정보 (제목, 저자 — 미선정 시 "미정" 표시)
- 발제문 목록 (미작성 시 "아직 발제문이 없습니다" 표시)
- 모임 기록 (미기록 시 "아직 기록이 없습니다" 표시)
- AI 요약 (녹음 내용 기반, 500자 이내 마크다운)
- 음성 녹음 시작/정지 버튼

### 4.3 모임원 관리 페이지 (모임장 전용)

- 현재 모임원 목록
- 모임원 추가 (이름 입력)
- 모임원 삭제

## 5. 데이터 구조 (Supabase)

### members 테이블
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| name | text | 이름 |
| role | text | 'leader' / 'member' |
| created_at | timestamp | 생성일 |

### schedule_proposals 테이블 (일정 제안)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| title | text | 제안 제목 (예: "1안: 5~8월 첫째 토요일 오후 3시") |
| description | text | 상세 설명 |
| proposed_by | uuid | 제안자 FK → members |
| created_at | timestamp | 생성일 |

### schedule_votes 테이블 (투표)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| proposal_id | uuid | FK → schedule_proposals |
| member_id | uuid | FK → members |
| vote | text | 'available' / 'unavailable' |
| created_at | timestamp | 생성일 |

### meetings 테이블 (확정된 모임)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| date | date | 모임 날짜 |
| time | text | 모임 시간 |
| status | text | 'confirmed' / 'completed' |
| proposal_id | uuid | FK → schedule_proposals |
| book_title | text | 도서 제목 |
| book_author | text | 도서 저자 |
| created_at | timestamp | 생성일 |

### discussion_items 테이블 (발제문/질문)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| meeting_id | uuid | FK → meetings |
| author_id | uuid | FK → members |
| type | text | 'topic' / 'question' |
| content | text | 발제문/질문 내용 |
| created_at | timestamp | 생성일 |

### meeting_records 테이블 (모임 기록)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| meeting_id | uuid | FK → meetings |
| content | text | 기록 내용 |
| audio_url | text | 음성 녹음 파일 URL (Supabase Storage) |
| ai_summary | text | AI 요약 (500자 이내, 마크다운) |
| created_at | timestamp | 생성일 |

### 데이터 저장 위치
- **구조화된 데이터**: Supabase PostgreSQL (테이블)
- **음성/영상 파일**: Supabase Storage (버킷)
- **배포**: Vercel

## 7. 음성 녹음 & AI 요약

### 음성 녹음
- 브라우저의 MediaRecorder API 사용
- 녹음 시작/정지/재생 기능
- 녹음된 파일은 Supabase Storage에 업로드

### AI 요약 (Gemini API)
- 모임 기록 텍스트를 Gemini API에 전달
- **500자 이내**의 마크다운 형식으로 구조적 요약 생성
- 요약 결과는 meetings_records 테이블에 저장
