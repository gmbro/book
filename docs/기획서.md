# 1+1 독서모임 - 서비스 기획서

> **배포 URL**: https://book-club-1plus1.vercel.app
> **최종 업데이트**: 2026년 3월 18일

---

## 1. 서비스 소개

**1+1 독서모임**은 10명의 모임원이 함께 일정을 잡고, 책을 고르고, 발제문을 쓰고, 모임 기록을 남기는 **올인원 독서모임 관리 서비스**입니다.

### 핵심 가치
| 가치 | 설명 |
|------|------|
| 🗓️ **빠른 결정** | 일정 투표로 모임 날짜를 빠르게 확정 |
| 📖 **체계적 관리** | 일정 → 도서 → 발제문 → 기록까지 한 흐름으로 관리 |
| 📱 **모바일 최적화** | 어디서든 스마트폰으로 편하게 참여 |
| 🌸 **감성적 디자인** | 봄 벚꽃 애니메이션과 부드러운 UI |

---

## 2. 사용자 흐름 (전체 Flow)

```
🏠 메인 화면 (로그인)
  │  이름 선택 → 생년월일 인증
  ▼
📋 일정 현황 페이지
  ├── 📅 달력 (제안/확정 일정 점 표시)
  ├── 🗳️ 일정 투표 카드 (참여/불참 + 댓글)
  ├── 📊 투표 카드 (장소/일정 투표)
  └── ⚙️ 설정 (생년월일 변경, 모임원 관리)
        │
        ▼  모임 클릭
📝 모임 상세 페이지
  ├── [도서] 도서 검색/선정 + 책 소개
  ├── [발제문] 발제문 작성 (인라인 에디터)
  └── [기록] 모임 기록 + 음성인식
```

## 3. 화면별 기능 상세

### 3.1 🏠 메인 화면 (로그인)

사용자가 앱에 처음 접속하면 보이는 화면입니다.

| 기능 | 상세 |
|------|------|
| 사용자 선택 | 10명의 이름 버튼 (셔플 표시) |
| 본인 인증 | 생년월일 6자리 (최초 1회 등록, 이후 로그인) |
| 모임장 표시 | 모임장 이름에 `모임장` 뱃지 |
| 배경 | 🌸 벚꽃 꽃잎이 떨어지는 봄 애니메이션 |

---

### 3.2 📋 일정 현황 페이지 (메인)

로그인 후 메인 페이지로, 모든 일정과 투표를 한눈에 볼 수 있습니다.

#### 📅 달력
- 월간 달력 형태로 일정 현황 표시
- 제안된 일정이 있는 날짜에 **보라색 점** 표시
- 확정된 일정은 **초록색 배경**으로 강조
- 확정 일정 날짜 클릭 → 모임 상세 페이지 이동
- **오늘** 버튼으로 현재 날짜로 바로 이동

#### 🗳️ 일정 투표 (안건 투표)
- 여러 안건에 대해 `참여 가능` / `불가능` 투표
- 각 안건별 투표 현황 (가능/불가/미참여 인원 이름 표시)
- **투표 마감 기한** 표시 (D-day)
- 미참여자에게 **알림 전송** (카카오톡 공유)
- 제안자만 자신의 안건 삭제 가능
- 모임장만 **일정 확정** 가능

#### 📊 투표 카드 (카카오톡 스타일)
- 장소 · 일정 기반 투표 생성
- `참여` / `미참여` 바(bar) 그래프로 시각화
- 투표별 **댓글** 기능 (의견 교환)
- 투표 마감 시 자동 마감 표시
- 생성자/모임장만 수정·삭제 가능

#### 다음 모임 섹션
- 확정된 모임 날짜/시간/도서 한눈에 표시
- 클릭하면 **모임 상세** 페이지로 이동
- 모임장: 수정/삭제/완료 처리 가능

#### ⚙️ 설정 (햄버거 메뉴)
- 생년월일 변경
- 모임원 관리 (모임장 전용)

---

### 3.3 📝 모임 상세 페이지

확정된 모임을 클릭하면 보이는 상세 페이지입니다. **도서 / 발제문 / 기록** 3개 탭으로 구성됩니다.

#### 📚 도서 탭
| 기능 | 상세 |
|------|------|
| 도서 검색 | Google Books API 연동, 제목/저자로 검색 |
| 도서 선정 | 검색 결과에서 선택하면 자동 등록 |
| 도서 정보 | 표지, 제목, 저자, 출판사, 출판년도 표시 |
| 별점 | ★ 별점 및 평가 수 표시 |
| 상세 정보 | 📖 페이지수, 카테고리 태그 표시 |
| 📝 책 소개 | 책 소개글 전체 표시 (누구나 열람 가능) |

#### ✏️ 발제문 탭
| 기능 | 상세 |
|------|------|
| 인라인 에디터 | 화면 전체를 차지하는 큰 입력창 |
| 저장 | 상단 `저장` 버튼으로 즉시 저장 |
| 목록 | 저장된 발제문 목록을 하단에 표시 |
| 수정/삭제 | 각 발제문별 수정·삭제 가능 |
| 전체 삭제 | 모든 발제문 일괄 삭제 |

#### 🎙️ 기록 탭 (클로바노트 스타일)
| 기능 | 상세 |
|------|------|
| 인라인 에디터 | 화면 전체를 차지하는 큰 입력창 |
| 🎤 음성 인식 | 마이크 버튼으로 한국어 음성→텍스트 변환 |
| 자동 입력 | 음성 인식 결과가 실시간으로 입력창에 반영 |
| 기록 저장 | 상단 `저장` 버튼으로 즉시 저장 |
| 기록 목록 | 저장된 기록은 하단에 작성자·시간과 함께 표시 |

---

### 3.4 👥 모임원 관리 (모임장 전용)

- 현재 모임원 목록 + 인원 수 표시
- 모임원 추가 (이름 입력)

## 7. 데이터 구조 (DB 테이블)

| 테이블 | 용도 |
|--------|------|
| `members` | 모임원 정보 (이름, 역할, 생년월일) |
| `schedule_proposals` | 일정 안건 (제목, 설명, 날짜 목록, 마감일) |
| `schedule_votes` | 일정 투표 (가능/불가능) |
| `meetings` | 확정된 모임 (날짜, 시간, 선정 도서) |
| `discussion_items` | 발제문 (작성자, 내용) |
| `meeting_records` | 모임 기록 (텍스트, AI 요약) |
| `polls` | 투표 카드 (제목, 설명, 마감일) |
| `poll_votes` | 투표 참여/미참여 |
| `poll_comments` | 투표 댓글 |
